What is a web app?

What is a web app?

  • Is used very often by several regular users.
  • Used on a variety of devices.
  • Users accomplish tasks or specific activities with it.

Nature of a web app

  • Works on several layouts, browsers, and bandwidth constraints.
  • Optimised to accomplish user's tasks on all these layouts and browsers.
  • Fast to load for frequent users, quick enough for new users.
  • Interface does not get in the way of user's intentions.

Design Constraints

  • Flexible enough to take advantage of all kinds of layouts.
  • Uses as few assets as possible.
  • Uses features as available on the platform it is run on.
  • Still presents a seamless experience for the user.

Which means…

  • More graphics you use, the more the assets.
  • More pixel perfection you yearn for, more the file sizes and the maintenance nightmare.
  • More polyfilling to torture older browsers to perform beyond their capability, slower is the user experience.
  • What could be the solution?

CSS3!

  • Present experience native to the user's browser/platform ability
  • Features that work on Opera 10.5+, IE9+, Firefox 3.6+, Safari 4+, Chrome 8+
  • Native animation trumps JavaScript animation
  • Dont use setTimeout and setInterval for visual effects.

  • No background images

    Less network requests, smaller files > greater performance
  • "Subtle CSS3 effects should be employed as a reward for users who run a modern browser."
    - Front-End Development Guidelines
  • Delight the user with whats available, and not moan about what isn't or make the experience terrible for those without.

Slideshow - Style

  position: relative;

Slideshow - Animations

.cssanimations .slideshow article {
/* Hide all slides to prepare
for display with animations */
  opacity: 0;

/* Each slide animates linearly
in 12s cycles forever using fade animation  */
  animation: fade 6s linear infinite;
}

.cssanimations .slideshow article:nth-of-type(2) {
/* Delay starting animation for this slide by 2s */
  animation-delay: 2s;
}

.cssanimations .slideshow article:last-of-type {
/* Delay starting animation for this slide by 4s */
  animation-delay: 4s;
}

Slideshow - Animations

/* The fade animation */
@keyframes fade {
/* Fades in from 0% to 4% of animation */
    0% { opacity: 0; }
    4% { opacity: 1; }

/* From 4% to 33% keep opacity steady at 1 */
   33% { opacity: 1; }

/* Fades out in 4%(37% - 31%) of animation */
   37% { opacity: 0; }
  100% { opacity: 0; }
}

Calculations

  • Each slide needs to stay on for 2 seconds.
  • Total time for 3 slides: 6 seconds
  • Each slide should be visible for about 2s…
  • …which translates to 33% (⅓) of the total animation time.

Easy Prefixed Gradient

  • Sass

    html {
  @include background-image(
  linear-gradient(left top, white, #ddd));
}

  • CSS

    html {
   background-image: -webkit-gradient(linear, 0% 0% , 100% 100% , color-stop( 0% , #ffffff ), color-stop( 100% , #dddddd ));
   background-image: -webkit-linear-gradient( left top , #ffffff , #dddddd );
   background-image: -moz-linear-gradient( left top , #ffffff , #dddddd );
   background-image: -ms-linear-gradient( left top , #ffffff , #dddddd );
   background-image: -o-linear-gradient( left top , #ffffff , #dddddd );
   background-image: linear-gradient( left top , #ffffff , #dddddd );
}

Remember

  • Prefixes: All or none*.
  • Always have fallback.
  • transparent = rgba(0, 0, 0, 0)
  • Webkit does not transform inline elements.
  • Transitions or animations on pseudo-elements (e.g. ::before, ::after) only available on Aurora.
  • Each gradient or transition needs to be specified 5 times for multiple prefixes (and prefixless).
  • Minimize use of CSS gradients, transforms & animations.
  • Use Modernizr, now in low-fat flavours: modernizr.github.com/Modernizr/2.0-beta/
